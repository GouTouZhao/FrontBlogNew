# 使用vue3

使用 npm run dev 启动

## 通用style css设计单列文件，如果确认此样式仅仅在这个文件内使用可以在文件内创建样式

## 对于元素、板块的设计参考dub文件夹，技术栈会有不同，你进行改为vue3复制进来即可

## 参考dub项目的亚克力，毛玻璃设计，卡片圆角等现代化设计

## 不要使用icron作为图标，直接使用文字代替即可，严禁使用带色的图案icron

## 昼夜切换黑白

## 设计风格就是现代化简约黑白

## 下滑时文字，板块等的出现仿照linear网站的文字虚化逐渐出现的效果

导航栏做在左侧，做可收缩设计，点开最下方是用户头像等个人信息，中间是分区，上方做一个昼夜切换按钮和展开/隐藏侧边栏按钮，当然仅在打开侧边栏时才能看到其他的所有侧边栏的东西

## 下滑的文字逐渐出现效果逐渐的时间延需要较长

## 白天的白底黑字没问题，但是夜间模式应该是黑底白字，还有对于卡片，白天直接用黑色的卡片增强对比，而夜间用白色卡片


## 每个分区打开刷新即get博客列表，默认第一页，默认不刷新，底部做分页，显示当前页数和上一页下一页，每页8条，空则显示：还没有发布任何博客，最后一页显示：我也是有底线的，侧边栏在进入分区的时候也同时在中间位置渲染这个列表，八个内容，上下面是省略号......表示上一页下一个，上一页和下一页不要使用箭头，直接使用省略号，前/后没有了就别渲染省略号了，导航栏也做直接显示四个分区，可以点击切换直接跳转到其分区的第一页，分区下滑也做逐渐显示效果


## 前端做区分各个页，比如start页是主页，其他各个分区就是其他页，链接上做/ 区分

## 前端发送到后端的地址是本机10002端口，使用axios做链接

## 如果是游客状态还没有登录，头像、名字都可以点击跳转到登录界面，登录页面不设置成单个页面，而是一个悬浮卡片样式，可以出现在任何界面，右上角带叉号点击关闭，登录完成后依旧是用户正在看的界面，不影响阅读

## 提醒的样式全部统一做成弹窗（比如提醒用户登录成功），在屏幕的正中间的上方，做成和背景相反颜色的弹窗，持续两秒，不带关闭按钮，可以触发多个，他们消失都是向上漂浮+渐隐，最新出现的在最上层，上层的会挡住下层，同时不影响底部的自动向上漂浮消失

## 对于登录/注册 返回的名字/双token/邮箱前端应该存在本地，其中refreshtoken存在httponly的cookie里，accesstoken存在localstorage里面，前端应该根据后端token刷新机制也进行设计回传刷新等机制，如果refresh token因为时间过期则提示用户重新登录，如果access token因为时间过期则自动刷新access token，这些机制能对应上后端的设计就可以

## 对齐打开分区的getarticlelist接口，这个文件里前面有提到过，现在不用那个前端的测试的几个blog例子了，直接使用和后端链接的真正数据，还有分页等也做好设计
